## Rule 112 — Boot Logging Audit

### What exists

`_recordSwBoot(reason)` fires on both `chrome.runtime.onInstalled` and
`chrome.runtime.onStartup`. Current captured fields per entry:

| Field | Source | Present |
|-------|--------|---------|
| `v` | `chrome.runtime.getManifest().version` | YES |
| `ts` | `new Date().toISOString()` | YES |
| `reason` | caller arg (`'startup'` / `'install'` / details.reason) | YES |

The ring-buffer cap is 50 entries, written to `gam_sw_boots` in
`chrome.storage.local`. The entire function is wrapped in `try/catch` (non-fatal).

### What is missing (Rule 112 gaps)

The following fields called for by Rule 112 are **absent** from the current boot
fingerprint:

1. **User-Agent** — `navigator.userAgent` is not captured. Chrome version, OS,
   and architecture are unrecoverable after the fact when diagnosing SW crashes.
2. **Tier** — the caller's mod tier (`workerModToken` present? lead?) is not
   logged. Boot context (privileged vs anonymous) is missing.
3. **boot_count** — no monotonic counter. The 50-entry ring lets you count
   backwards, but a named `boot_count` field allows zero-diff queries.
4. **Storage usage** — `chrome.storage.local.getBytesInUse` is called in the
   weekly maintenance alarm (line 583) and the diagnostic ping (line 460) but
   NOT in `_recordSwBoot`. A boot fingerprint without storage pressure is
   incomplete: bloated storage is a leading indicator of SW instability.
5. **Console log is sparse** — current log line:
   `[modtools vX.Y.Z] SW boot at <ts> reason=startup`
   Missing: UA, tier, and bytes-in-use are not emitted even to the console,
   making live debugging weaker than the stored record.

### Proposed comprehensive boot fingerprint

```js
async function _recordSwBoot(reason) {
  try {
    const ver        = chrome.runtime.getManifest().version;
    const ua         = navigator.userAgent;
    const ts         = new Date().toISOString();

    // Tier probe: check secretCache or fall back to storage
    let tier = 'unknown';
    try {
      const r = await chrome.storage.local.get('gam_settings');
      const s = (r && r.gam_settings) || {};
      if (s.isLeadMod)          tier = 'lead';
      else if (s.workerModToken) tier = 'mod';
      else                       tier = 'anon';
    } catch (_) {}

    // Storage pressure
    let bytesInUse = -1;
    try {
      bytesInUse = await new Promise(
        (res, rej) => chrome.storage.local.getBytesInUse(null,
          n => chrome.runtime.lastError ? rej(chrome.runtime.lastError) : res(n)
        )
      );
    } catch (_) {}

    // Monotonic boot counter
    const raw       = await chrome.storage.local.get('gam_sw_boots');
    const boots     = Array.isArray(raw && raw.gam_sw_boots) ? raw.gam_sw_boots : [];
    const bootCount = boots.length + 1; // simple 1-based from this entry onward

    const entry = { v: ver, ts, reason: reason || 'unknown', ua, tier, bytesInUse, boot_count: bootCount };
    boots.push(entry);
    if (boots.length > 50) boots.splice(0, boots.length - 50);
    await chrome.storage.local.set({ gam_sw_boots: boots });

    console.log(
      `[modtools v${ver}] SW boot #${bootCount} at ${ts}` +
      ` reason=${reason} tier=${tier} storage=${bytesInUse}B ua=${ua}`
    );
  } catch (e) { /* non-fatal */ }
}
```

**Net additions:** UA, tier probe, bytes-in-use, boot_count, richer console line.
Storage read cost is negligible — `getBytesInUse` is already called on every
weekly alarm tick.

## Rule 113 — Telemetry (Opt-In) Design

### Existing telemetry infrastructure

- Worker has a `MOD_METRICS` Analytics Engine binding (v9.4.7, fixed).
  No client-side crash telemetry feeds it today.
- `_recordNetCall` exists in modtools.js and logs net events locally, but
  nothing ships those events to the worker.
- No `gam_settings.telemetry_enabled` key exists anywhere in the codebase.

### Proposed opt-in crash telemetry design

**Privacy-first defaults confirmed:** `telemetry_enabled` defaults `false`.
No PII, no username, no post content. Anon crash events only.

#### Client-side: global error sink (`background.js`)

```js
// AF-38 Rule 113: opt-in anon crash telemetry
async function _maybeReportCrash(errType, fileAndLine, message) {
  try {
    const r  = await chrome.storage.local.get('gam_settings');
    const s  = (r && r.gam_settings) || {};
    if (!s.telemetry_enabled) return;          // opt-in gate
    const token = secretCache && secretCache.workerModToken;
    if (!token) return;                        // no token -> no write
    const ver = chrome.runtime.getManifest().version;
    const payload = {
      event_type: 'crash',
      error_type: String(errType).slice(0, 64),
      file_line:  String(fileAndLine).slice(0, 128),
      message:    String(message).slice(0, 256),
      version:    ver,
      ts:         new Date().toISOString(),
    };
    // Fire-and-forget; never throw outward
    fetch(WORKER_BASE + '/telemetry/crash', {
      method: 'POST',
      headers: { 'content-type': 'application/json', 'x-mod-token': token },
      body: JSON.stringify(payload),
    }).catch(() => {});
  } catch (_) {}
}

// Wire to unhandledrejection + error
self.addEventListener('unhandledrejection', ev => {
  const e = ev.reason;
  _maybeReportCrash(
    (e && e.constructor && e.constructor.name) || 'UnhandledRejection',
    'background.js:unhandledrejection',
    (e && e.message) || String(e)
  ).catch(() => {});
});
self.addEventListener('error', ev => {
  _maybeReportCrash(
    (ev.error && ev.error.constructor && ev.error.constructor.name) || 'Error',
    (ev.filename || 'background.js') + ':' + ev.lineno,
    ev.message || ''
  ).catch(() => {});
});
```

#### Worker side (`/telemetry/crash` route, worker already has MOD_METRICS binding)

```js
// POST /telemetry/crash
// Validates token, writes one data point to MOD_METRICS Analytics Engine.
// Fields: event_type, error_type, file_line, message, version, ts.
// Rate-limited: 1 write per (token, error_type, file_line) per 5 minutes
// to prevent crash-storm flooding.
```

#### Popup UI (`popup.js` — settings panel)

```js
// AF-38 Rule 113: telemetry toggle
// Read initial state
const { gam_settings } = await chrome.storage.local.get('gam_settings');
const telEnabled = !!(gam_settings && gam_settings.telemetry_enabled);
document.getElementById('telemetryToggle').checked = telEnabled;

document.getElementById('telemetryToggle').addEventListener('change', async (e) => {
  const cur = await chrome.storage.local.get('gam_settings');
  const s   = { ...(cur.gam_settings || {}), telemetry_enabled: e.target.checked };
  await chrome.storage.local.set({ gam_settings: s });
});
```

Popup HTML addition (settings section):

```html
<label class="gam-setting-row">
  <input type="checkbox" id="telemetryToggle">
  <span>Send anonymous crash reports (opt-in)</span>
</label>
<p class="gam-setting-note">No usernames, no post content. Error type, file:line, and version only.</p>
```

**Key design constraints satisfied:**
- Default OFF (privacy-first).
- No PII in payload.
- Token-gated (anonymous users cannot write).
- Fire-and-forget, never throws outward.
- Worker rate-limits to absorb crash storms.

### Summary table

| Verdict | Count |

### Remediation for the one CONCERN

In `modtools.js` line 11294, the auto-crawl catch should at minimum log the
auth failure to the console rather than eating it:

```js
} catch(e) {
  // AF-38 Rule 114: don't swallow auth errors silently
  if (e && e.message && e.message.includes('mod token')) {
    console.warn('[ModTools] background crawl skipped:', e.message);
  }
}
```

No user-visible alert is warranted here (this fires silently on every page
load) but a `console.warn` makes the auth failure visible in DevTools without
requiring an active debug session.
