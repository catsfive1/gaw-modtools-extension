## Verified method notes (carry forward)

- **This file has CRLF line endings**, AND several bar-button glyphs inside template-literal strings are stored as literal `\uXXXX` ESCAPE-SEQUENCE TEXT, not the rendered Unicode character (confirmed via `od -c` byte dump) — inconsistently, since some `title` attributes on the SAME line use the real rendered character. Edit-tool `old_string` matching against the wrong form (glyph vs. escape-text) fails silently with "not found." **What works:** narrow `old_string` to the smallest unique ASCII anchor; the Edit tool's own write path auto-normalizes whatever you put in `new_string` to the file's existing escape-sequence convention on save (verified via `grep`/`od` after the fact).
- **For large multi-hundred-line block replacements**, the Edit tool gets expensive/fragile. Switch to a Node splice script: `readFileSync` → `indexOf` two ASCII-only anchors → verify uniqueness (`firstIndexOf === lastIndexOf`) → `writeFileSync`. Byte-exact by construction, sidesteps CRLF/escape traps entirely.
- **`brave://extensions/` and `chrome://extensions/` are NOT reachable via the Claude-in-Chrome `navigate` tool** — it prepends `https://` regardless of existing scheme, producing a broken URL (`https://brave//extensions/`) that leaves the tab on an error page. Confirmed twice, with two different schemes. Extension reload is a genuine Commander-only manual action (`brave://extensions` → reload icon → F5 the tab) — do not keep retrying automation for it.
- **Boot-crash probe** (`scratchpad/tdz_probe.mjs` — recreate if the scratchpad is gone; it's a small stubbed-global harness that `eval`s the content-script IIFE and catches any synchronous top-level throw) is the single highest-value verification step after any edit to `modtools.js`. `node --check` only catches syntax errors; this catches the actual `ReferenceError`/TDZ class of bug that has bitten this codebase repeatedly (v10.6.1 `FEATURE_FLAGS`, v10.36.1 `MSG_QUEUE_KEY`, v10.36.6 `mc`). Run it on both the repo file AND the loaded `dist\mod-tools dist\modtools.js` after every build.
- **Ship loop, every version, no exceptions:** bump `manifest.json` → add a `CHANGELOG.md` entry above the previous top entry → `node --check modtools.js` (and `background.js`/`modtools-aux.js` if touched) → run the FULL `scripts/_*.mjs` suite → run the boot-crash probe → `git add` the specific changed files (never `-A`) → commit (standing authorization, no need to ask) → `git push` → `pwsh -File scripts/build-zip.ps1 -NoPause` (rebuilds `dist\mod-tools dist\`, the folder Brave actually loads) → re-probe the LOADED file → `pwsh -File scripts/mirror-to-drive.ps1 -Version X.Y.Z -NoPause`. **Commit ≠ shipped** — the build step is not optional; skipping it means Commander stays on the old version no matter how much you committed.
- **Test suite convention:** `scripts/_*.mjs`, zero npm dependencies (no jsdom) — slice the real function out of `modtools.js`/`background.js` via `SRC.indexOf(startMarker)` / `SRC.indexOf(endMarker, startFromHere)` (always pass the second `fromIndex` arg — CRLF and repeated comment text have caused false-marker matches more than once this session), `new Function(...)` it with minimal hand-rolled stubs, assert real behavior. Currently **19 files, 271 passing assertions.** Every new fix or feature gets a new numbered test file (`_p0_...` through `_p6_...` so far, Sonnet's parallel work uses its own naming).
