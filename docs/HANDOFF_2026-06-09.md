Confirmed by reading the code ‚Äî the current build recovers a token with NO user action:
- **`loadSecrets()` (background.js:388)** runs on `onInstalled` AND `onStartup` (survives eviction):
  session ‚Üí local (plaintext-first) ‚Üí **L5 backup self-heal** (decrypt-independent backup).

So a service-worker eviction is a **non-event on v10.28.0+**. Residual lockout vectors are rare +
user-initiated: full data wipe or server-side token rotation ‚Äî both ~20s to recover.
