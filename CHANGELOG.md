## v10.14.x — Polish + Migration Cycle + Auto-Unsticky War (2026-05-10 → 2026-05-11)

**v10.14.4 AUTO-UNSTICKY CS-SCAN ARCHITECTURE** (`12b665f`, 2026-05-11) — The v10.10.0 worker-cron design (scrape GAW homepage every 5 min) has been completely non-functional in production since ship. v10.14.3 surfaced the threshold-sync gap; **v10.14.3 diagnostic deploy** (worker version `85f78d57`) then captured the smoking gun via audit chain rows: every `*/5` cron fire received HTTP 403 with `cf-ray` header + `<title>Attention Required!` body. **Cloudflare Bot Fight Mode rejects worker-to-worker fetches with a JS challenge interstitial that workers cannot solve** (no DOM, no JS exec on the response). Curl from a non-CF host with the same UA returned HTTP 200, confirming worker-egress-specific not header-specific. Architectural fix: scrape stickies from the CS where real browsers solve CF challenges trivially. NEW worker endpoint `POST /admin/auto-unsticky-scan` (worker version `afca9131-3ba2-4567-81c8-523a10f090b5`) — lead-only ingest that takes a `stickies[]` array, applies the same `team_settings` threshold check the cron uses, de-dupes against pending/claimed `auto_action_queue` rows, inserts qualifying candidates with `reason='auto_unsticky:cs_scan_age_and_votes'`. NEW `background.js` RPC `autoUnstickyScanReport` forwards CS calls with `asLead` lead-token auth. NEW `modtools.js` CS scanner IIFE (~60 lines at end of file): GAW homepage only (`pathname='/'`), `isLeadMod()` gate, 5-min throttle via `chrome.storage.local.gam_last_auto_unsticky_scan_at`, scrapes `.post.sticky[data-id]` elements, extracts (thingId/votes/ageH/createdAt/title), POSTs candidates, snacks `'Auto-unsticky: N candidate(s) queued (lead-side scan)'` on hits. Existing mod-side queue execution path (SW polls `auto_action_queue`, claims, CS unstickies via authenticated GAW cookies) unchanged. Diagnostic enhancement retained on cron path (logs `cf-ray` + body on 403). **5th "shipped-but-not-wired" instance found in this v10.13/v10.14 effort** (v10.13 RALPH AUDIT meta-pattern continues to apply).
